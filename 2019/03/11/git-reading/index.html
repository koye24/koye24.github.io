git的基本使用

1.在要使用git管理的项目的 根目录内 进行 git仓库的初始化

2.验证是否初始化成功

3.输入 git status 查看状态

4.添加到暂存区 git add 文件名 添加所有文件 git add .

5.执行 提交到本地仓库 git commit -m 备注信息

6.重新查看 状态 git status 出现以下信息 表示 git成功使用了 放心的下班了

git忽略清单

在git项目中，有些文件是不需要进行共享 比如 个人开发者的密码。。 或者是 node_modules 里面一堆小文件。。

想要在git仓库中 忽略某些文件 不希望这些文件 被git管理

  1. 在根目录下 新建一个文件 git 忽略清单 .gitignore

  2. 直接在文件内 加入 要忽略的文件名就可以了 支持正则语法 忽略 pwd.txt

分支

分支提供了一种更加高效的管理不同项目版本的方式

  1. 在git仓库中 一直就存在分支的概念 默认是在主分支 master下工作

  2. 在git仓库中 开启n个分支，正常操作的话，分支之间的编辑 互不影响

  3. 不同的分支之间的关系 就好像不同的文件夹的关系一样

过程
1.开启新分支

2.查看本地仓库的分支

3.切换分支

5.不同的分支有它自己的仓库，对文件进行编辑之后一样需要提交到本地仓库

6.合并分支
​ 1.必须要先切换会主分支 master
​ 2.再进行合并
